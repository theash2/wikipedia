<div class="para" label-module="para"><b>GNU</b>是一个自由的<a target=_blank href="/item/%E6%93%8D%E4%BD%9C%E7%B3%BB%E7%BB%9F">操作系统</a>，其内容软件完全以<a target=_blank href="/item/GPL">GPL</a>方式发布。这个操作系统是<a target=_blank href="/item/GNU%E8%AE%A1%E5%88%92">GNU计划</a>的主要目标，名称来自GNU's Not Unix!的<a target=_blank href="/item/%E9%80%92%E5%BD%92%E7%BC%A9%E5%86%99">递归缩写</a>，因为GNU的设计类似Unix，但它不包含具著作权的Unix代码。GNU的创始人，<a target=_blank href="/item/%E7%90%86%E6%9F%A5%E5%BE%B7%C2%B7%E9%A9%AC%E4%BF%AE%C2%B7%E6%96%AF%E6%89%98%E6%9B%BC">理查德·马修·斯托曼</a>，将GNU视为“达成社会目的技术方法”。</div><div class="para" label-module="para">作为操作系统，GNU的发展仍未完成，其中最大的问题是具有完备功能的内核尚未被开发成功。GNU的内核，称为<a target=_blank href="/item/Hurd">Hurd</a>，是<a target=_blank href="/item/%E8%87%AA%E7%94%B1%E8%BD%AF%E4%BB%B6%E5%9F%BA%E9%87%91%E4%BC%9A">自由软件基金会</a>发展的重点，但是其发展尚未成熟。在实际使用上，多半使用<a target=_blank href="/item/Linux%E5%86%85%E6%A0%B8">Linux内核</a>、<a target=_blank href="/item/FreeBSD">FreeBSD</a>等替代方案，作为系统核心，其中主要的操作系统是Linux的发行版。<a target=_blank href="/item/Linux">Linux</a>操作系统包涵了<a target=_blank href="/item/Linux%E5%86%85%E6%A0%B8">Linux内核</a>与其他自由软件项目中的GNU组件和软件，可以被称为<a target=_blank href="/item/GNU%2FLinux">GNU/Linux</a>（见<a target=_blank href="/item/GNU%2FLinux%E5%91%BD%E5%90%8D%E4%BA%89%E8%AE%AE">GNU/Linux命名争议</a>）。</div>

</div>“GNU”是“GNU's Not Unix!”（GNU并非Unix！）的首字母递归缩写。<sup class="sup--normal" data-sup="1" data-ctrmap="2Db3aKdextt:1,">
[1]</sup><a class="sup-anchor" name="ref_[1]_36272">&nbsp;</a>
Stallman宣布GNU应当发音为Guh-NOO以避免与new这个单词混淆（注：gnu在英文中原意为非洲牛羚，发音与new相同）。</div><div class="para" label-module="para">UNIX是一种广泛使用的商业操作系统的名称。由于GNU将要实现UNIX系统的接口标准，因此GNU计划可以分别开发不同的操作系统部件。GNU计划采用了部分当时已经可自由使用的软件，例如TeX排版系统和X Window视窗系统等。不过GNU计划也开发了大批其他的自由软件。</div><div class="anchor-list ">

</div><div class="para-title level-2  J-chapter" data-index="2" label-module="para-title">
<h2 class="title-text"><span class="title-prefix">GNU</span>历史</h2>
<a class="edit-icon j-edit-link" data-edit-dl="2" href="javascript:;"><em class="cmn-icon wiki-lemma-icons wiki-lemma-icons_edit-lemma"></em>编辑</a>
<a class="audio-play part-audio-play J-part-audio-play" href="javascript:;"><em class="cmn-icon wiki-lemma-icons wiki-lemma-icons_voice-play"></em>
<span class="J-part-audio-text">播报</span>
</a>
</div>
<div class="para" label-module="para">1985年Richard Stallman又创立了<a target=_blank href="/item/%E8%87%AA%E7%94%B1%E8%BD%AF%E4%BB%B6%E5%9F%BA%E9%87%91%E4%BC%9A">自由软件基金会</a>（Free Software Foundation）来为GNU计划提供技术、法律以及财政支持。尽管GNU计划大部分时候是由个人自愿无偿贡献，但FSF有时还是会聘请程序员帮助编写。当GNU计划开始逐渐获得成功时，一些商业公司开始介入开发和技术支持。当中最著名的就是之后被Red Hat兼并的 Cygnus Solutions。</div><div class="para" label-module="para"><div class="lemma-picture J-lemma-picture text-pic layout-right" 
	 
		style="width:180px; float: right;"
	>
<a class="image-link" nslog-type="9317"

		style="width:180px;height:144px;"
		>
<img  class="lazy-img" src="data:image/png;base64,iVBORw0KGgoAAAANSUhEUgAAAAEAAAABCAMAAAAoyzS7AAAAGXRFWHRTb2Z0d2FyZQBBZG9iZSBJbWFnZVJlYWR5ccllPAAAAAZQTFRF9fX1AAAA0VQI3QAAAAxJREFUeNpiYAAIMAAAAgABT21Z4QAAAABJRU5ErkJggg==" data-src="https://bkimg.cdn.bcebos.com/pic/cc11728b4710b9120913c4b2c3fdfc03934522af?x-bce-process=image/resize,m_lfit,w_360,limit_1/format,f_auto"  alt="GNU操作系统的内核Linux"

<span class="description">
GNU操作系统的内核Linux
</span>
</div>到了1990年，GNU计划已经开发出的软件包括了一个功能强大的文字编辑器<a target=_blank href="/item/Emacs">Emacs</a>。<a target=_blank href="/item/GCC/17570" data-lemmaid="17570">GCC</a>（GNU Compiler Collection，GNU<a target=_blank href="/item/%E7%BC%96%E8%AF%91%E5%99%A8">编译器</a>集合），是一套由 GNU 开发的编程语言编译器。以及大部分UNIX系统的<a target=_blank href="/item/%E7%A8%8B%E5%BA%8F%E5
3890
%BA%93">程序库</a>和工具。唯一依然没有完成的重要组件就是操作系统的<a target=_blank href="/item/%E5%86%85%E6%A0%B8">内核</a>(称为HURD)。</div><div class="para" label-module="para">1991年<a target=_blank href="/item/Linus%20Torvalds">Linus Torvalds</a>编写出了与UNIX兼容的Linux操作系统内核并在GPL条款下发布。Linux之后在网上广泛流传，许多程序员参与了开发与修改。1992年Linux与其他GNU软件结合，完全自由的操作系统正式诞生。该操作系统往往被称为“GNU/Linux”或简称Linux。（尽管如此GNU计划自己的内核Hurd依然在开发中，已经发布<a target=_blank href="/item/Beta%E7%89%88%E6%9C%AC">Beta版本</a>。）</div><div class="para" label-module="para">许多UNIX系统上也安装了GNU软件，因为GNU软件的质量比之前UNIX的软件还要好。GNU工具还被广泛地移植到Windows和Mac OS上。</div><div class="para" label-module="para">GNU 包含3个协议条款，</div><div class="para" label-module="para">GPL：<a target=_blank href="/item/GNU%E9%80%9A%E7%94%A8%E5%85%AC%E5%85%B1%E8%AE%B8%E5%8F%AF%E8%AF%81/393832" data-lemmaid="393832">GNU通用公共许可证</a>（GNU General Public License）</div><div class="para" label-module="para">LGPL：GNU较宽松公共许可证 (GNU Lesser General Public License）, ) ，旧称 GNU Library General Public License (GNU 库通用公共许可证)；</div><div class="para" label-module="para">GFDL ： <a target=_blank href="/item/GNU%E8%87%AA%E7%94%B1%E6%96%87%E6%A1%A3%E8%AE%B8%E5%8F%AF%E8%AF%81">GNU自由文档许可证</a>（GNU Free Documentation License ）的缩写形式。</div><div class="para" label-module="para"><div class="lemma-picture J-lemma-picture text-pic layout-right" 
	 
		style="width:220px; float: right;"
	>
<a class="image-link" nslog-type="9317"

		>

							style="width:220px;height:333.33333333333px;"
					/>
</a>
<span class="description">
Linux内核的缔造者Linus Torvalds
</span>
</div>这里指的自由，并不是价格免费，这和价格无关而是使用软件对所有的用户来说是自由的。GPL通过如下途径实现这一目标：</div><div class="para" label-module="para">1.它要求软件以<a target=_blank href="/item/%E6%BA%90%E4%BB%A3%E7%A0%81">源代码</a>的形式发布，并规定任何用户能够以源代码的形式将软件复制或发布给别的用户。</div><div class="para" label-module="para">2.如果用户的软件使用了受 GPL 保护的任何软件的一部分，那么该软件就继承了 GPL 软件，并因此而成为 GPL 软件，也就是说必须随应用程序一起发布源代码。</div><div class="para" label-module="para">3.GPL 并不排斥对自由软件进行商业性质的包装和发行，也不限制在自由软件的基础上打包发行其他非自由软件。</div><div class="para" label-module="para">由于GPL很难被<a target=_blank href="/item/%E5%95%86%E4%B8%9A%E8%BD%AF%E4%BB%B6">商业软件</a>所应用，它要求调用它的库的代码也得GPL，全部开放，并且一同发布，不能直接连接。所以后来GNU推出了LGPL许可证</div><div class="para" label-module="para">在GPL与LGPL许可证保护下发布<a target=_blank href="/item/%E6%BA%90%E4%BB%A3%E7%A0%81">源代码</a>的结果很相似，对旧代码所做的任何修改对于想知道这些代码的人必须是公开的，唯一真正的不同之处在于私人版权代码是否可以与<a target=_blank href="/item/%E5%BC%80%E6%94%BE%E6%BA%90%E4%BB%A3%E7%A0%81">开放源代码</a>相互连接，LGPL允许实体连接私人代码到开放源代码，并可以在任何形式下发布这些合成的<a target=_blank href="/item/%E4%BA%8C%E8%BF%9B%E5%88%B6%E4%BB%A3%E7%A0%81">二进制代码</a>。只要这些代码是动态连接的就没有限制。（使用<a target=_blank href="/item/%E5%8A%A8%E6%80%81%E9%93%BE%E6%8E%A5">动态链接</a>时，即使是程序在运行中调用函数库中的函数时，应用程序本身和函数库也是不同的实体）</div><div class="anchor-list ">
<a name="3" class="lemma-anchor para-title" ></a>
<a name="sub36272_3" class="lemma-anchor " ></a>
<a name="许可证" class="lemma-anchor " ></a>
</div><div class="para-title level-2  J-chapter" data-index="3" label-module="para-title">
<h2 class="title-text"><span class="title-prefix">GNU</span>许可证</h2>
<a class="edit-icon j-edit-link" data-edit-dl="3" href="javascript:;"><em class="cmn-icon wiki-lemma-icons wiki-lemma-icons_edit-lemma"></em>编辑</a>
<a class="audio-play part-audio-play J-part-audio-play" href="javascript:;"><em class="cmn-icon wiki-lemma-icons wiki-lemma-icons_voice-play"></em>
<span class="J-part-audio-text">播报</span>
</a>
</div>
<div class="para" label-module="para">GNU 认为任何软件都应当以 <a target=_blank href="/item/%E8%87%AA%E7%94%B1%E8%BD%AF%E4%BB%B6">自由软件</a> 发布。要让一个软件成为<a target=_blank href="/item/%E8%87%AA%E7%94%B1%E8%BD%AF%E4%BB%B6">自由软件</a>，您需要把它以自由软件<a target=_blank href="/item/%E8%AE%B8%E5%8F%AF%E8%AF%81">许可证</a>发布。我们通常使用 GNU 通用公共许可证 (<a target=_blank href="/item/GNU%20GPL">GNU GPL</a>) 发布软件，但有时我们也使用 其它自由软件许可证 发布软件。在 GNU 软件包中，GNU 只使用与GNU GPL兼容的许可证。</div><div class="para" label-module="para">自由软件的文档应当是 自由文档 ，以便人们可以随着软件的改进去更新它或是重新发布它。若要把文档以自由文档的形式发布，您需要使用自由文档许可证。我们通常使用 GNU 自由文档许可证 (GNU FDL) ，但少数情况下我们也使用 其它的自由文档许可证 。</div><div class="para" label-module="para">如果您开展了一个新的项目，但不太确定应该选用何种许可证，那么 “如何为您的作用选择一个许可证” 一文可以以一种简单且容易遵循的方式给您以帮助。我们正在进一步修订我们的文档许可证，同时欢迎听到您对提案的建议。请访问 许可证更新网站 阅读最新的草案并参与到修订过程中。<sup class="sup--normal" data-sup="2" data-ctrmap=":2,">
[2]</sup><a class="sup-anchor" name="ref_[2]_36272">&nbsp;</a>
</div><div class="anchor-list ">

</div><div class="para-title level-2  J-chapter" data-index="4" label-module="para-title">
<h2 class="title-text"><span class="title-prefix">GNU</span>组成</h2>
<a class="edit-icon j-edit-link" data-edit-dl="4" href="javascript:;"><em class="cmn-icon wiki-lemma-icons wiki-lemma-icons_edit-lemma"></em>编辑</a>
<a class="audio-play part-audio-play J-part-audio-play" href="javascript:;"><em class="cmn-icon wiki-lemma-icons wiki-lemma-icons_voice-play"></em>
<span class="J-part-audio-text">播报</span>
</a>
</div>
<div class="para" label-module="para">主条目：GNU软件包列表</div><div class="para" label-module="para">该系统的基本组成包括<a target=_blank href="/item/GNU%E7%BC%96%E8%AF%91%E5%99%A8/10444865" data-lemmaid="10444865">GNU编译器</a>套装（<a target=_blank href="/item/GCC">GCC</a>）、GNU的C库（<a target=_blank href="/item/glibc">glibc</a>）、以及GNU核心工具组（<a target=_blank href="/item/coreutils">coreutils</a>），另外也是GNU除错器（<a target=_blank href="/item/GDB">GDB</a>）、GNU二进制实用程序（binutils）的GNU Cashshell中和GNOME桌面环境。GNU开发人员已经向GNU应用程序和工具的Linux<a target=_blank href="/item/%E7%A7%BB%E6%A4%8D">移植</a>，也广泛应用在其它操作系统中使用，如<a target=_blank href="/item/BSD">BSD</a>变体的<a target=_blank href="/item/Solaris">Solaris</a>，和<a target=_blank href="/item/OS%20X">OS X</a>作出了贡献。</div><div class="para" label-module="para">许多GNU程序已经被移植到其他操作系统，包括<a target=_blank href="/item/%E4%B8%93%E6%9C%89%E8%BD%AF%E4%BB%B6">专有软件</a>，如<a target=_blank href="/item/Microsoft%20Windows">Microsoft Windows</a>和OS X.GNU计划已经被证明是比他们的专有Unix更为可靠。截至2015年11月，全国共有466 GNU软件包（包括退役，但不包括383 ）主办的官方GNU开发的网站。</div><div class="anchor-list ">
<a name="5" class="lemma-anchor para-title" ></a>
<a name="sub36272_5" class="lemma-anchor " ></a>
<a name="GNU 变种" class="lemma-anchor " ></a>
</div><div class="para-title level-2  J-chapter" data-index="5" label-module="para-title">
<h2 class="title-text"><span class="title-prefix">GNU</span>GNU 变种</h2>
<a class="edit-icon j-edit-link" data-edit-dl="5" href="javascript:;"><em class="cmn-icon wiki-lemma-icons wiki-lemma-icons_edit-lemma"></em>编辑</a>
<a class="audio-play part-audio-play J-part-audio-play" href="javascript:;"><em class="cmn-icon wiki-lemma-icons wiki-lemma-icons_voice-play"></em>
<span class="J-part-audio-text">播报</span>
</a>
</div>
<div class="para" label-module="para">主条目：GNU variants</div><div class="para" label-module="para">GNU项目的官方核心是GNU Hurd；然而，截至2012年，Linux核心正式以<a target=_blank href="/item/Linux-libre">Linux-libre</a>为名成为GNU项目中的一部分，并删除了所有专有组件的变种<sup class="sup--normal" data-sup="3" data-ctrmap=":3,">
[3]</sup><a class="sup-anchor" name="ref_[3]_36272">&nbsp;</a>
。</div><div class="para" label-module="para">像FreeBSD内核其他核心依然和GNU软件合作，形成一个工作的操作系统。FSF认为Linux核心，与GNU工具和实用程序使用时，应承认自己为GNU变种，并促进改成<i>GNU/Linux</i>的这类系统名称（导致<a target=_blank href="/item/GNU%2FLinux%E5%91%BD%E5%90%8D%E4%BA%89%E8%AE%AE">GNU/Linux命名争议</a>） 。<a target=_blank href="/item/GNU%E5%B7%A5%E7%A8%8B/9610051" data-lemmaid="9610051">GNU工程</a>已认可的Linux发行版，如<a target=_blank href="/item/gNewSense">gNewSense</a>，<a target=_blank href="/item/TRISQUEL">TRISQUEL</a>和<a target=_blank href="/item/Parabola%20GNU%2FLinux-libre">Parabola GNU/Linux-libre</a>。不使用Hurd作为核心的GNU的其他变种包括的Debian GNU / kFreeBSD和Debian GNU / NetBSD的，将得以对BSD的核心实现GNU的早期计划。</div>
